## 节点一：/go/share（分享创造）

**标题：**
```
做了个免费追剧资源导航，每天自动检测资源是否还活着，开源的，求反馈
```

**正文：**
```
追剧踩坑踩多了，发现每次找资源都要先确认"这个站今天还能用吗"，然后大概率发现挂了，再换下一个，再挂……

于是做了个导航：awesome-zhuiju-free

核心就一件事：GitHub Actions 每天早上自动跑一遍，检测里面每个资源能不能访问，结果直接标在 README 里。不能用的标红，访问受限的标黄。

收录了在线影视、影视APP、网盘搜索、磁力BT、字幕、TVBox/影视仓配置地址、IPTV订阅源这几个类别，目前 [62] 个资源。

几个设计选择解释一下：

1. 数据全部在 resources.json 里，有 schema 验证，PR 合进来会自动跑检测。不想做成一个黑盒的导航站。

2. 资源有评分维度（多/快/净/稳/易）和风险标注（版权/安全/隐私/支付），追剧的资源基本都有版权问题，就如实标 high 了，不掩盖。

3. 有 Issue 模板，普通用户也能直接提交新资源，维护者审核后用命令自动发布，不需要懂 Git。

上线一周多，GitHub 上涨到了 [1000]+ Star，说明这个需求是真实存在的。

有几个问题想请教 V 友：
- 还有哪些类型的追剧资源值得收录？
- 现在 README 的信息量够不够清晰？有没有看不懂的地方？
- 有没有人愿意一起维护，特别是对某个分类比较熟悉的？

GitHub：https://github.com/laoma2053/awesome-zhuiju-free
网站：https://zhuiju.me
```

## 节点二：/go/movie（电影）

**标题：**
```
整理了一份免费无广告的追剧资源清单，每天自动检测能不能用，开源
```

**正文：**
```
找免费资源经常遇到的情况：
- 收藏的网站打不开了
- 打开有一堆广告和弹窗
- 点了半天播不了

做了个开源项目收集这些资源，特点是每天自动检测一遍所有链接是否可访问，能用的标绿，挂了的标红，不用自己一个个去试。

分类：在线影视 / 影视APP / 网盘搜索 / 磁力BT / 字幕 / TVBox配置 / IPTV订阅源

[62] 个资源，持续更新中。完全免费，无广告，开源。

GitHub：https://github.com/laoma2053/awesome-zhuiju-free
国内直接访问：https://zhuiju.me

如果有好用的资源欢迎在 GitHub 提 Issue 补充，有维护者会审核。
```

## 发帖注意事项

- **发 /go/share 时**：要准备好回复评论，V2EX 用户会追问技术细节，及时回复是关键
- **不要同一天两个节点都发**：间隔一天，避免被认为是刷量
- **不要用"求 Star"等词**：V2EX 反感，删掉显得功利的表达
- **"开源"和"无广告"要反复出现**：这是V2EX 用户最在意的两点
- **主动承认缺点**：比如说"版权问题如实标了 high，不掩盖"，反而会加分
